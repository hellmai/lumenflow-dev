## Why LumenFlow?

<CardGrid stagger>
  <Card title="Clarity over chaos" icon="pencil">
    Every Work Unit has explicit acceptance criteria. No ambiguity about what "done" means.
  </Card>
  <Card title="AI-first design" icon="rocket">
    Built for AI agents to understand and execute. Clear specs, automated gates, context recovery.
  </Card>
  <Card title="Quality by default" icon="approve-check">
    Gates (format, lint, typecheck, test) run automatically. No "works on my machine."
  </Card>
  <Card title="Flow, not queues" icon="forward-slash">
    Lane-based WIP limits prevent overload. Finish work before starting new work.
  </Card>
</CardGrid>

## Give your agent one link

LumenFlow is designed for agent-first setup. Give your agent this link and it will install the CLI,
initialize LumenFlow, create the first WU, and stop.

```text
Set up LumenFlow for this repo using the Agent Quickstart:
https://lumenflow.dev/getting-started/quickstart/
```

Works with Claude Code, Cursor, Codex, Windsurf, Cline, and any AI that can read docs.

## How it works

```yaml
# 1. Define your lanes
lanes:
  - name: 'Framework: Core'
    wip_limit: 1
  - name: 'Experience: UI'
    wip_limit: 2

# 2. Create a Work Unit (ID auto-generated)
# pnpm wu:create --title "Add auth" --lane "Framework: Core" \
#   --type feature --exposure api \
#   --description "Add login and session handling" \
#   --acceptance "Users can log in with email/password" \
#   --code-paths "src/auth/" \
#   --test-paths-unit "src/auth/__tests__/auth.test.ts" \
#   --plan

# 3. Claim and work (use generated ID, example: WU-123)
# pnpm wu:claim --id WU-123 --lane "Framework: Core"
# cd worktrees/framework-core-wu-123

# 4. Run gates in worktree, then complete from main
# pnpm wu:prep --id WU-123
# cd /path/to/main && pnpm wu:done --id WU-123
```
